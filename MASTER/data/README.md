# MASTER Data Directories

This directory contains generated and runtime data for the MASTER framework.

## Directory Structure

### `memory/`
Vector-based memory storage. Files are saved/loaded by `lib/memory.rb`.
- Format: YAML or JSON
- Content: Chunked content with embeddings and metadata

### `intelligence/`
Harvested ecosystem intelligence from OpenClaw and other sources.
- Generated by: `lib/harvester.rb`
- Format: YAML files with datestamp (`harvested_YYYY-MM-DD.yml`)
- Content: Skill metadata, trends, quality signals

### `monitoring/`
LLM usage tracking and cost monitoring.
- Generated by: `lib/monitor.rb`
- Format: JSONL (JSON Lines)
- File: `usage.jsonl`
- Content: Per-task token counts, costs, durations

### `reports/`
Automated weekly reports combining all intelligence gathering.
- Generated by: `bin/weekly`
- Format: Markdown
- Files: `weekly_YYYY-MM-DD.md`
- Content: Harvest results, optimization status, monitoring summary

## Notes

- All files in these directories are **generated** and should not be committed to version control
- Files are excluded via `.gitignore`
- Safe to delete; will be regenerated as needed
- Weekly automation keeps history growing; prune old files periodically
